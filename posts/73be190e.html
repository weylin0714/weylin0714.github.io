大数据学习环境搭建(三)--Hive单机环境搭建

Hive
大数据
发布日期:   2020-02-03
更新日期:   2020-03-07
文章字数:   1.7k
阅读时长:   7 分
阅读次数:  

Hive是一个架构在Hadoop之上的数据仓库工具，可以将结构化的数据文件映射成为一张数据库表，并提供类SQL的查询功能，为大数据查询和分析提供方便。这里在阿里云服务器上搭建Hive单机伪分布环境。

1. Hive安装与配置

1.1 下载与安装

下载Hive安装包，这里我下载版本为 cdh5.16.2。下载地址：http://archive.cloudera.com/cdh5/cdh/5/

  • 将安装包移动到/usr/local/src目录下，并解压到/usr/local/目录下，重命名为hive
tar -zxvf hive-1.1.0-cdh5.16.2.tar.gz -C /usr/local/
# 将解压到/usr/local/目录下的文件夹重命名
mv hive-1.1.0-cdh5.16.2 hive
  • 配置环境变量
# 打开配置文件 
vim /etc/profile 
# 添加Hive环境变量 
export HIVE_HOME=/usr/local/hive
export PATH=$PATH:$HIVE_HOME/bin
  • 环境变量生效
source /etc/profile

1.2 MySQL数据库安装

采用MySQL数据库保存Hive的元数据，而不采用Hive自带的derby来存储元数据。

1.2.1 安装前准备

  • 下载并安装MySQL yum仓库

可以到MySQL Yum 仓库下载页选择合适的版本

# 下载rpm包
wget http://repo.mysql.com/mysql57-community-release-el7-11.noarch.rpm
# 安装rpm包获得MySQL yum仓库
sudo rpm -ivh mysql57-community-release-el7-11.noarch.rpm
  • 选择一个发行版

修改配置文件，选择MySQL5.7版本

# 修改/etc/yum.repos.d/目录下的mysql-community.repo配置文件，启用想安装的MySQL版本，这里我们选择MySQL5.7
vi /etc/yum.repos.d/mysql-community.repo
# 将配置文件中MySQL 5.7 Community Server的enabled属性值为1，其余都为0
  • 验证mysql5.7是否已启用
[root@centos yum.repos.d]# yum repolist enabled | grep mysql
mysql-connectors-community/x86_64 MySQL Connectors Community                 141
mysql-tools-community/x86_64      MySQL Tools Community                      105
mysql57-community/x86_64          MySQL 5.7 Community Server                 404

1.2.2 安装MySQL

安装MySQL，并自动安装依赖包

 yum install mysql-community-server

1.2.3 启动MySQL

启动MySQL服务并检查状态

# 启动MySQL
systemctl start mysqld.service
#检查状态
systemctl status mysqld.service

1.2.4 修改MySQL密码

MySQL默认创建了‘root’@’localhost’账号，并创建了临时密码，密码打印在mysql日志中，找到并修改密码

# 获取临时密码
grep 'temporary password' /var/log/mysqld.log
# 用获取的密码登录MySQL
mysql -uroot -p
# 登陆MySQL后，修改密码。密码规则：至少包含1个大写字母、1个小写字母、1个数字、1个特殊字符，总长度不小于8
ALTER USER 'root'@'localhost' IDENTIFIED BY '新密码';

1.2.5 Navicat连接MySQL

  • 打开服务器的3306端口
  • Navicat连接报错：Host ‘IP’ is not allowed to connect to this MySQL server，解决方法：
# 使用账号密码登陆MySQL
mysql -uroot -p
# 选择数据库
use mysql;
# 允许任何主机连接
update user set host='%' where user='root';
# 刷新权限
flush privileges;

1.2.6 创建hive数据库

在MySQL中创建一个数据库，用来保存hive元数据

# 创建hive数据库，进入到mysql命令行
mysql> create database hive;
# 将hive数据库的字符编码设置为latin1
mysql> alter database hive character set latin1;

1.3 Hive配置

1.3.1 修改配置

  • hive-env.sh

进入/usr/local/hive/conf目录，拷贝Hive的环境配置模板flume-env.sh.template

cp hive-env.sh.template hive-env.sh

修改hive-env.sh，指定Hadoop的安装路径

HADOOP_HOME=/usr/local/hadoop
  • hive-site.xml

在/usr/local/hive/conf目录下，新建 hive-site.xml 文件，主要是配置存放元数据的MySQL的地址、驱动、用户名和密码等信息：

# 编辑hive-site.xml文件
vim hive-site.xml
<?xml version="1.0" encoding="UTF-8" standalone="no"?>
<?xml-stylesheet type="text/xsl" href="configuration.xsl"?>
<configuration>
  <property>
    <name>javax.jdo.option.ConnectionURL</name>
    <value>jdbc:mysql://localhost:3306/hive?createDatabaseIfNotExist=true&amp;useSSL=false</value>
    <description>JDBC connect string for a JDBC metastore</description>
  </property>
  <property>
    <name>javax.jdo.option.ConnectionDriverName</name>
    <value>com.mysql.jdbc.Driver</value>
    <description>Driver class name for a JDBC metastore</description>
  </property>
  <property>
    <!--MySQL用户名-->  
    <name>javax.jdo.option.ConnectionUserName</name>
    <value>USERNAME</value>
    <description>username to use against metastore database</description>
  </property>
  <property>
    <!--MySQL密码-->  
    <name>javax.jdo.option.ConnectionPassword</name>
    <value>PASSWORD</value>
    <description>password to use against metastore database</description>
  </property>
</configuration>

1.3.2 添加数据库驱动

由于Hive在连接MySQL时需要JDBC驱动，所以首先需要下载对应版本的驱动，然后将驱动移动到/usr/local/hive/lib中。

# 解压
tar -zxvf mysql-connector-java-5.1.48.tar.gz
# 进入到解压目录，将mysql-connector-java-5.1.48.jar移动到/usr/local/hive/lib目录下
cd mysql-connector-java-5.1.48
cp mysql-connector-java-5.1.48.jar /usr/local/hive/lib/

1.3.3 初始化元数据库

  • 当使用的 hive 是 1.x 版本时，可以不进行初始化操作，Hive 会在第一次启动的时候会自动进行初始化，但不会生成所有的元数据信息表，只会初始化必要的一部分，在之后的使用中用到其余表时会自动创建。
  • 当使用的 hive 是 2.x 版本时，必须手动初始化元数据库。初始化命令：
# schematool 命令在/usr/local/hive/bin目录下
schematool -dbType mysql -initSchema

这里我使用的是 CDH 的hive-1.1.0-cdh5.16.2.tar.gz，对应 Hive 1.1.0 版本，可以跳过这一步。

1.3.4 启动

启动hive之前，请先启动hadoop集群

# 启动hive
[root@centos ~]# hive

2. HiveServer2/beeline配置

Hive内置了HiveServer2服务，允许远程客户端可以使用各种编程语言向 Hive 提交请求并检索结果，支持多客户端并发访问和身份验证。HiveServer2 拥有自己的 CLI 工具——Beeline。Beeline 是一个基于 SQLLine 的 JDBC 客户端。

2.1 修改Hadoop配置

修改 /usr/local/hadoop/etc/hadoop/目录下hadoop 的 core-site.xml 配置文件，增加如下配置，指定 hadoop 的 root 用户可以代理本机上所有的用户。

<property>
    <name>hadoop.proxyuser.root.hosts</name>
    <value>*</value>
</property>
<property>
    <name>hadoop.proxyuser.root.groups</name>
    <value>*</value>
</property>

2.2 启动hiveserver2

进入到/usr/local/hive/bin目录下，执行以下命令：

[root@centos bin]# hiveserver2
# 打开另一个命令窗口，执行以下命令，出现10000的端口号表示成功开启
netstat -nltp | grep 10000
[root@centos bin]# netstat -nltp | grep 10000
tcp        0      0 0.0.0.0:10000           0.0.0.0:*               LISTEN      17080/java

hivesever2每次要手动启动，因此设置为进程，后台运行。

[root@centos ~]# nohup hive --service hiveserver2 &

2.3 beeline的使用

进入/usr/local/hive/bin目录下，执行以下命令进入beeline 交互式命令行，连接hive

[root@centos bin]# beeline -u jdbc:hive2://localhost:10000 -n root
# -u : 指定元数据库的链接信息  -n : 指定用户名

还有一种方式也可以去连接

# 先执行 beeline
[root@centos bin]# beeline
beeline>
# 再执行以下命令进行连接
!connect jdbc:hive2://localhost:10000 -n root
