命令行解析--sys.argv与模块argparse

argparse argv

在执行Python脚本时，有时我们希望通过命令行传入参数。在本文中，我们介绍命令行参数解析 sys.argv 和 argparse 的使用方法。

1. sys.argv

argv 是 sys 模块中的方法，用于获取命令行参数列表。在执行Python脚本时，可以从命令行向脚本传入参数。

先看一个例子：创建一个demo.py文件

import sys

arg1 = sys.argv[0]
args = sys.argv[:]
print(arg1)
print(args)

在命令行执行 python demo.py 4 5 6，输出：

demo.py
['demo.py', '4', '5', '6']

可以看到 sys.argv 可以访问到所有命令行传入的参数，返回值是一个列表。

  • sys.argv[0]：执行时传入的脚本名，在我们示例中是 demo.py。

  • sys.argv[:]：获取所有的参数。由于 sys.argv 返回是一个列表，我们可以使用列表索引或切片操作，获取传入的单个或多个参数，比如 sys.argv[1] 为 '4'，sys.argv[1:] 结果是 ['4', '5', '6']。

再看一个示例，从外部将参数传入函数：

import sys

def add_demo(*args):
    num = sum(*args)
    print(num)

if __name__ == '__main__':
    lst = map(lambda x:int(x), sys.argv[1:]) 
    add_demo(lst)

执行 python demo.py 4 5 6，输出 15。

2. argparse

2.1 示例

argparse 是 Python 中自带的一个命令行解析模块。使用整个模块主要有以下几步：

  • 导入模块：import argparse
  • 创建 ArgumentParser 对象：parser = argparse.ArgumentParser()
  • 添加参数：parser.add_argument()
  • 解析参数：parser.parse_args()

首先看一个示例，新建一个 prog.py 文件：

import argparse

# 创建 ArgumentParser 对象
parser = argparse.ArgumentParser(description='Process some integers.')
# 添加参数
parser.add_argument('integers', metavar='N', type=int, nargs='+',
                    help='an integer for the accumulator')
parser.add_argument('--sum', dest='accumulate', action='store_const',
                    const=sum, default=max,
                    help='sum the integers (default: find the max)')
# 解析参数
args = parser.parse_args()
print(args.accumulate(args.integers))

在命令行执行 python prog.py -h，可以查看帮助信息：

usage: prog.py [-h] [--sum] N [N ...]

Process some integers.

positional arguments:
  N           an integer for the accumulator

optional arguments:
  -h, --help  show this help message and exit
  --sum       sum the integers (default: find the max)

在命令行分别执行 python prog.py 1 2 3 4 和 python prog.py 1 2 3 4 --sum，分别输出 4 和 10。

如果不加 –sum，默认输出传入整数的最大值；如果传入 –sum，则输出传入整数的和。这里通过示例演示 argparse 模块使用的几个步骤，对于其中各参数的含义这里不必深究，在下面会具体介绍。

2.2 命令行参数类型

在介绍 argparse 模块中各函数调用前，先要了解一下常见的命名命令行参数类型：

  • 位置参数（positional arguments）：有具体的参数名，参数前无 - 或 -- 等前缀，而且是必须配置的参数，如示例中的 N。
  • 可选参数（optional arguments）：参数前加 - 或 -- 等前缀，如示例中的 --sum。可选参数对于代码运行不是必须的，但是设置后可以改变程序的行为。

命令行参数通用格式为：program [optional] positional。

2.3 ArgumentParser()

ArgumentParser 对象包含将命令行解析成 Python 数据类型所需的全部信息。创建一个新的 ArgumentParser对象时，所有的参数都应当作为关键字参数传入。下面具体介绍各参数的含义：

  • prog

程序的名称，默认情况下，sys.argv[0] 作为程序的名称。可以通过设置 prog 参数自定义程序名称。

parser = argparse.ArgumentParser(prog='myprogram')
parser.print_help()

输出：

usage: myprogram [-h]

optional arguments:
  -h, --help  show this help message and exit
  • usage

描述程序用途的字符串，默认情况下，ArgumentParser 根据它包含的参数来构建用法消息。可以通过 usage= 关键字参数覆盖这一默认消息。

parser = argparse.ArgumentParser(prog='PROG', usage='%(prog)s [options]')
parser.add_argument('--foo', nargs='?', help='foo help')
parser.print_help()

输出：

usage: PROG [options]

optional arguments:
  -h, --help   show this help message and exit
  --foo [FOO]  foo help
  • description

在参数帮助文档之前显示的文本，通过 description= 设置。

parser = argparse.ArgumentParser(description='This is the description.')
parser.print_help()

输出：

usage: prog.py [-h]

This is the description.

optional arguments:
  -h, --help  show this help message and exit
  • epilog

在参数帮助文档之后显示的文本，通过 epilog= 设置。

  • prefix_chars

    可选参数的前缀字符集合，默认是 _。如果想使用其他的前缀符，可通过 prefix_chars = 设置。

  • fromfile_prefix_chars

当处理的命令行参数非常长时，把它存入一个文件而不是直接在命令行输入是个比较好的选择。从文件读取的参数在默认情况下必须一个一行，并且它们被视为与命令行上的原始文件引用参数位于同一位置。

with open('args.txt', 'w') as fp:
    fp.write('-f\nbar')
parser = argparse.ArgumentParser(fromfile_prefix_chars='@')
parser.add_argument('-f')
args = parser.parse_args()
print(args)
# 命令行执行 python prog.py @args.txt，输出
Namespace(f='bar')

还有其他参数设置，这里不再一一介绍，详情可访问官网查看。

2.4 add_argument()

定义单个的命令行参数应当如何解析。下面具体介绍一些参数的含义。

  • name or flags

一个命名或者一个选项字符串的列表，例如 foo 或 -f, --foo。

parser = argparse.ArgumentParser(prog='PROG')
parser.add_argument('-f', '--foo')
parser.add_argument('bar')
args1 = parser.parse_args(['BAR'])
args2 = parser.parse_args(['BAR', '--foo', 'FOO'])
print(args1)
print(args2)
# 输出
Namespace(bar='BAR', foo=None)
Namespace(bar='BAR', foo='FOO')

注意，这里是直接将参数列表 ['BAR', '--foo', 'FOO'] 传入 parse_args()，也可以不将参数列表传入parse_args()，在命令行执行时传入，如 python prog.py 'BAR' '--foo' 'FOO'。下面为了方便，都是直接将参数列表传入，并打印输出返回的对象。

  • action

指定了命令行参数应当如何处理。action 有以下几个选项：

store：存储参数的值。这是默认的动作。

store_const：存储被 const 命名参数指定的值，该动作常用于可选参数来指定一些标志。

parser = argparse.ArgumentParser()
parser.add_argument('--foo', action='store_const', const=42)
args = parser.parse_args(['--foo'])
print(args)
# 输出
Namespace(foo=42)

store_true 和 store_false：是 'store_const' 分别用作存储 True 和 False 值的特殊用例，默认值分别为 False 和 True。

parser = argparse.ArgumentParser()
parser.add_argument('--foo', action='store_true')
parser.add_argument('--bar', action='store_false')
parser.add_argument('--baz', action='store_false')
args = parser.parse_args('--foo --bar'.split())
print(args)
# 输出
Namespace(bar=False, baz=True, foo=True)

append：将参数值存储到一个列表中，对于一个可选参数设置多次时很有用。

parser = argparse.ArgumentParser()
parser.add_argument('--foo', action='append')
args = parser.parse_args('--foo 1 --foo 2'.split())
print(args)
# 输出
Namespace(foo=['1', '2'])

append_const：存储一个列表，并将 const 命名参数指定的值追加到列表中。一般在多个参数需要在同一列表中存储常数时会有用。

parser = argparse.ArgumentParser()
parser.add_argument('--str', dest='types', action='append_const', const=str)
parser.add_argument('--int', dest='types', action='append_const', const=int)
args = parser.parse_args('--str --int'.split())
print(args)
# 输出
Namespace(types=[<class 'str'>, <class 'int'>])

count：统计一个关键字参数出现的数目或次数。

parser = argparse.ArgumentParser()
parser.add_argument('--verbose', '-v', action='count', default=0)
args = parser.parse_args(['-vvv'])
print(args)
# 输出
Namespace(verbose=3)

nargs：关联不同数目的命令行参数到单一动作。支持以下几个选项：

  • N（一个整数）：命令行中的 N 个参数会被聚集到一个列表中。
parser = argparse.ArgumentParser()
parser.add_argument('--foo', nargs=2) 
parser.add_argument('bar', nargs=1)
args = parser.parse_args('c --foo a b'.split())
print(args)
# 输出
Namespace(bar=['c'], foo=['a', 'b'])
  • ？：一个命令行参数最多接收一个参数值。如果没有对应的参数，则使用 default 值。对于可选参数，有一个特例-只显示了可选参数，没有跟随一个值，则使用 const 值。
parser = argparse.ArgumentParser()
parser.add_argument('--foo', nargs='?', const='c', default='d')
parser.add_argument('bar', nargs='?', default='d')
args1 = parser.parse_args(['XX', '--foo', 'YY'])
args2 = parser.parse_args(['XX', '--foo'])
args3 = parser.parse_args([])
print(args1)
print(args2)
print(args3)
# 输出
Namespace(bar='XX', foo='YY')
Namespace(bar='XX', foo='c')
Namespace(bar='d', foo='d')
  • *：所有当前命令行参数被聚集到一个列表中。
parser = argparse.ArgumentParser()
parser.add_argument('--foo', nargs='*')
parser.add_argument('--bar', nargs='*')
parser.add_argument('baz', nargs='*')
args = parser.parse_args('a b --foo x y --bar 1 2'.split())
print(args)
# 输出
Namespace(bar=['1', '2'], baz=['a', 'b'], foo=['x', 'y'])
  • +：和 '*' 类似，所有当前命令行参数被聚集到一个列表中。另外，当一个命令行参数都没有指定时，会抛出一个错误信息。
parser = argparse.ArgumentParser(prog='PROG')
parser.add_argument('foo', nargs='+')
args1= parser.parse_args(['a', 'b'])
print(args1)
args2 = parser.parse_args([])
print(args2)
# 输出
Namespace(foo=['a', 'b'])
usage: PROG [-h] foo [foo ...]
PROG: error: the following arguments are required: foo
  • type

默认情况下，ArgumentParser 对象将命令行参数当作简单字符串读入。当需要传入其他类型时，可通过 type= 指定。

parser = argparse.ArgumentParser()
parser.add_argument('foo', type=int)
  • metavar

在帮助消息中该参数对应的名称。

parser = argparse.ArgumentParser()
parser.add_argument('--foo', metavar='YYY')
parser.add_argument('bar', metavar='XXX')
parser.parse_args('X --foo Y'.split())
parser.print_help()

输出：

usage: prog.py [-h] [--foo YYY] XXX

positional arguments:
  XXX

optional arguments:
  -h, --help  show this help message and exit
  --foo YYY

注意 metavar 仅改变显示的名称。
