1. 关系数据库与表

1. 1 关系数据库结构

关系数据库是应用非常广泛的数据库，和Excel工作表类似，采用由行和列组成的二维表来管理数据。用来关系数据库的计算机系统称为关系数据库管理系统（RDBMS）。RDBMS常见的系统结构是客户端/服务器类型（C/S）：

1.2 表的结构

关系数据库中的数据表为二维表，表的列称为字段，表的行称为记录。关系数据库必须以行为单位进行数据读写。

2. SQL概述

SQL：操作数据库的语言，用关键字、表名、列名等组合而成的语句来描述操作的内容。SQL语句根据功能可以分为：

  • 数据定义语言（Data Definition Language, DDL）：创建或删除数据库或表。
CREATE # 创建数据库和表
DROP   # 删除数据库和表
ALTER  # 修改数据库和表结构
  • 数据操作语言（Data Manipulation Language, DML）：查询或修改表中的记录。
SELECT # 查询表中的数据
INSERT # 向表中插入数据
UPDATE # 更新表中的数据
DELETE # 删除表中的数据
  • 数据控制语言（Data Control Language, DCL）：用来确认或取消对数据库中数据进行的变更。
COMMIT   # 确认对数据库中数据的变更
ROLLBACK # 取消对数据库中数据的变更
GRANT    # 赋予用户操作权限
REVOKE   # 取消用户的操作权

SQL基本的书写规则：

  • SQL语句要以分号（;）结尾。
  • SQL语句关键字不区分大小写。

3. 表的创建与更新

3.1 创建表

在创建表之前要先创建数据库：CREATE DATABASE <数据库名称>

CREATE DATABASE shop;

创建数据表：

CREATE TABLE <表名>
(<列名1> <数据类型> <列约束>,
 <列名1> <数据类型> <列约束>,
 ...
<表的约束1>, <表的约束2>);
--示例
CREATE TABLE Product
(product_id      CHAR(4)      NOT NULL,
 product_name    VARCHAR(100) NOT NULL,
 product_type    VARCHAR(32)  NOT NULL,
 sale_price      INTEGER ,
 purchase_price  INTEGER ,
 regist_date     DATE ,
 PRIMARY KEY (product_id));
  • 数据库、表与字段的命名规则：只能使用字母、数字、下划线命名，且必须以字母开头。同一个数据库中表名不能相同，同一个表中字段名也不能相同。
  • 字段数据类型：表中所有列必须指定数据类型。
    • INTEGER型：指定存储整数的列的数据类型。
    • CHAR型：用来指定存储字符串的列的数据类型。CHAR(N)，N可存储字符串的最大长度。字符串以定长字符串形式存储，即当字符串长度不到N时，自动用空格补足。
    • VARCHAR型：指定存储字符串的列的数据类型。VARCHAR(N)，字符串以可变长字符串形式存储，即当字符串长度不足时，不自动补足。
  • 约束设置：NOT NULL约束表示字段不能为空，PRIMARY KEY()将指定列设为主键。

3.2 更新表

--添加多列
ALTER TABLE <tbl_name> ADD COLUMN (col_name1 col_type, col_name2 col_type, ...);
ALTER TABLE <tbl_name> ADD (col_name1 col_type, col_name2 col_type, ...);
ALTER TABLE <tbl_name> ADD col_name1 col_type, ADD col_name2 col_type, ...);
--修改多列
ALTER TABLE <tbl_name> change col_name1 col_name3 col_type, change col_name2 col_name4 col_type;
ALTER TABLE <tbl_name> change COLUMN col_name1 col_name3 col_type, change COLUMN col_name2 col_name4 col_type;
--修改字段名
ALTER TABLE <tbl_name> change `original_field_name` changed_field_name field_type;
--删除多列
ALTER TABLE <tbl_name> DROP col_name1, DROP col_name2;
ALTER TABLE <tbl_name> DROP COLUMN col_name1, DROP COLUMN col_name2;

3.3 删除表

DROP TABLE <tbl_name>;

3.4 重命名表

不同的DBMS对表重命名的支持有所不同，在Mysql中使用RENAME进行操作。

RENAME TABLE <original_tbl_name> <renamed_tbl_name>

4. 数据更新

4.1 数据插入

使用INSERT语句向表中插入数据，基本语法：

INSERT INTO <表名> (列1, 列2, ...) VALUES (值1, 值2, ...)

将列名和值用逗号隔开，分别括在 ( ) 内，这种形式称为清单。列名组成的为列清单，值组成的为值清单。原则上，执行一次INSERT语句会插入一行数据。

目前，大多数RDBMS都支持一次插入多行数据，如在Mysql中插入多行数据：

INSERT INTO <表名> VALUES (val1, val2, ...),
                        (val3, val4, ...);

4.1.1 省略列清单

当对表的全列进行INSERT操作时，可以省略列清单，这时VALUES子句的值默认按照从左到右的顺序赋值给每一列。

4.1.2 插入NULL

如果想给某一列赋值NULL，可直接在值清单中写入NULL，前提是相应列不能设置NOT NULL约束。

4.1.3 插入默认值

如果在CREATE创建表时设置了DEFAULT约束，在我们INSERT操作时可以插入默认值。

CREATE TABLE Product
(product_id      CHAR(4)      NOT NULL,
 product_name    VARCHAR(100) NOT NULL,
 product_type    VARCHAR(32)  NOT NULL,
 sale_price      INTEGER      DEFAULT 0,
 purchase_price  INTEGER ,
 regist_date     DATE ,
 PRIMARY KEY (product_id));

插入默认值有显式和隐式两种：

  • 显式方法：在VALUES子句中指定DEFAULT关键字。
INSERT INTO <tbl_name> (col_name1, col_name2, ...) VALUES (val1, DEFAULT, ...)
  • 隐式方法：在列清单和值清单中省略默认值的列。
INSERT INTO <tbl_name> (col_name1, col_name3, ...) VALUES (val1, val3, ...)

实际使用时，建议使用显式方法。

4.1.4 从其他表复制数据

INSERT INTO <tbl_name> (col1, col2, ...)
SELECT col1, col2, ...
FROM <tbl_name1>

INSERT语句的SELECT语句中，可以使用WHERE子句或GROUP BY子句等。

4.2 数据更新

UPDATE <tbl_name>
    SET <col_name1> = <expression1>,
        <col_name2> = <expression2>,
        <col_name3> = NULL,
        ...
    WHERE <condition_expr>;
  • 当更新多列时，将SET后的列=值间用逗号分隔。
  • 使用WHERE指定筛选条件，这种指定更新对象的UPDATE语句为搜索型UPDATE语句。
  • 当将列更新为NULL时，直接将列赋值为NULL。

4.3 数据删除

DELETE FROM <tbl_name>
 WHERE <condition>;

可以通过WHERE子句指定对象条件来删除部分数据，如果不使用WHERE指定删除条件，则删除表的全部行。

5. 数据查询基础

5.1 SELECT查询基础

5.1.1 查询部分列数据

SELECT col_name1, col_name2, ...
FROM tbl_name;

5.1.2 查询表的所有列

SELECT *
FROM tbl_name;

5.1.3 为列设置别名

SELECT col_name AS alias
FROM <tbl_name>;

别名可以使用中文，需要使用双引号（” “）括起来。

5.1.4 查询不重复的行

SELECT DISTINCT col_name1, col_name2, ...
FROM <tbl_name>;

使用DISTINCT删除重复的行，只能用在第一个列名之前，但是作用于后面所有的列。在使用DISTINCT时，NULL也被视为一类数据。

5.1.5 返回部分记录

SELECT *
FROM <tbl_name>
# 第一种方式
LIMIT m OFFSET n;   # m为查询的行数，n为偏移量，即从第几行开始查询
# 第二种方式
LIMIT n, m;  # n为查询结果的索引值(默认从0开始)，当i=0时可省略；m为查询结果返回的数量。LIMIT 5 等价于 LIMIT 0, 5

5.2 数据过滤

很多时候我们并不需要查询出全部数据，只需要通过设定一些过滤条件来查询部分数据。

5.2.1 使用WHERE子句

SELECT <列名>, ...
FROM <表名>
WHERE <条件表达式>;

首先通过WHERE子句查询出符合指定条件的记录，然后再选取出SELECT指定的列。注意：WHERE子句要紧跟在FROM子句之后。条件表达式中：

  • 大小比较：可以使用>、>=、<、<=、<>、!=等操作符。
  • 范围值检查：使用BETWEEN...AND...。
  • 空值检查：使用IS NULL, IS NOT NULL。

在实际应用时，条件表达式可能不仅仅只有一个条件，这时可以使用组合WHERE子句，即以AND子句或OR子句的方式使用。

  • AND操作符

WHERE子句中，使用AND连接多个条件，用来查询满足所有给定条件的记录。

  • OR操作符

使用OR操作符，用来检索匹配任一给定条件的记录。

  • 求值顺序

WHERE子句中也可以同时包含任意数目的AND和OR操作符，允许两者结合进行复杂、高级的过滤。注意：任何时候使用具有AND和OR操作符的WHERE子句，都应该使用圆括号明确地进行分组。

5.2.2 IN 操作符

IN是OR的简便用法，用来指定条件范围，范围内的每个条件都可以进行匹配。

示例：筛选出多个价格的记录

# 使用OR
SELECT product_name, product_price
FROM Products
WHERE product_price = 150
OR product_price = 180
OR product_price = 200;
# 使用IN
SELECT product_name, product_price
FROM Products
WHERE product_price IN (150, 180, 200);

如果想筛选不包含指定条件时，可使用NOT IN来实现。

5.2.3 NOT操作符

用于否定NOT后所跟的任何条件。

SELECT product_name, product_price
FROM Products
WHERE NOT product_price = 250;

5.2.4 LIKE操作符

当需要进行字符串部分一致查询时可使用LIKE操作符。

  • 前方一致
SELECT *
FROM <tbl_name>
WHERE col_name LIKE 'STR%';  # 匹配字段col_name以字符串STR开头的记录
  • 中间一致
SELECT *
FROM <tbl_name>
WHERE col_name LIKE '%STR%';  # 匹配字段col_name包含字符串STR的记录
  • 后方一致
SELECT *
FROM <tbl_name>
WHERE col_name LIKE '%STR';  # 匹配字段col_name以字符串STR结束的记录

5.3 聚合与排序

5.3.1 聚合查询

使用聚合函数对表中数据进行操作或计算。常用的5各聚合函数：COUNT、SUM、AVG、MAX、MIN。

  • COUNT：统计表中记录的行数
COUNT(*) # 统计表中所有的记录数，包含字段值为NULL的记录
COUNT(1) # 统计表中所有的记录数，包含字段值为NULL的记录
COUNT(<列名>) # 统计指定列的记录数，除字段值为NULL的记录
COUNT(DISTINCT <列名>) # 统计指定列的不同值的记录数，会忽略字段值为NULL的记录

如果表中没有主键，count(1)效率优于count(*)；如果表中存在主键，count(主键列名)效率最优；如果列为主键，count(列名)效率优于count(1)；如果列不为主键，count(1)效率优于count(列名)；如果表中只有一列，则count(*)效率最优。

  • SUM：计算合计值
SELECT SUM(<col_name>)
FROM <tbl_name>;

SUM聚合函数会将NULL排除在外，即如果待统计的字段col_name中有NULL，则不会被统计。

  • AVG：计算多行数据的平均值
SELECT AVG(<col_name>)
FROM <tbl_name>;

AVG也会将NULL排除在外。

  • MAX/MIN：计算多条记录中的最大值/最小值
SELECT MIN(<col_name1>), MAX(<col_name2>)
FROM <tbl_name>;

MAX/MIN函数几乎适用于所有数据类型的列，而SUM/AVG函数只适用于数值类型的列。

5.3.2 分组

SELECT <列名1>, <列名2>, ...
FROM <表名>
WHERE <条件>
GROUP BY <列名1>, <列名2>, ...;

GROUP BY 子句中指定的列为聚合键或分组列。使用GROUP BY的子句书写顺序：

SELECT --> FROM --> WHERE --> GROUP BY

会先根据WHERE子句指定的条件进行过滤，再进行汇总处理。SELECT语句的执行顺序：

FROM --> WHERE --> GROUP BY --> SELECT

在使用GROUP BY 时，有些常见错误需要注意：

  • 在SELECT子句中有多余的列：SELECT子句中不能出现聚合键之外的列名，
  • 在GROUP BY子句中不能使用SELECT子句中定义的别名：主要是由于执行顺序决定的，先执行GROUP BY，再执行SELECT。
  • GROUP BY子句结果的显示是无序的。
  • GROUP BY子句中不能使用聚合函数，只有SELECT子句、HAVING子句和ORDER BY子句中可以使用聚合函数。

如果在使用GROUP BY分组后，想进一步过滤分组，可以使用HAVING。

SELECT <列名1>, <列名2>, ...
FROM <表名>
WHERE <条件>
GROUP BY <列名1>, <列名2>, ...
HAVING <分组结果对应的条件>;

示例：从按照商品种类分组后的结果中，取出数量大于2的分组

SELECT product_type, COUNT(*)
FROM Product
GROUP BY product_type
HAVING COUNT(*) > 2;

在HAVING子句中可以使用以下三种要素：常数、聚合函数、GROUP BY子句中指定的列名（即聚合键）。

WHERE和HAVING子句中都可以指定筛选条件，但是有区别的：

  • WHERE子句中是指定行所对应的条件
  • HAVING子句中式指定组所对应的条件

GROUP BY子句中的聚合键所对应的条件不应写在HAVING子句中，而应写在WHERE子句中。

5.3.3 排序

SELECT <列名1>, <列名2>, ...
FROM <表名>
WHERE <条件>
GROUP BY <列名1>, <列名2>, ...
HAVING <分组结果对应的条件>
ORDER BY <排序基准列1>, <排序基准列2> DESC;

ORDER BY子句后的列名为排序键，默认为升序排序，可通过DESC指定为降序排序。如果排序键中包含NULL，则会在开头或末尾进行汇总。此外，在ORDER BY子句中可以使用SELECT子句中未使用的列和聚合函数。

以上基本介绍了常用的子句，完整的书写顺序如下：

SELECT子句 --> FROM子句 --> WHERE子句 --> GROUP BY子句 --> HAVING子句 --> ORDER BY子句

粗略的执行顺序（不同的DBMS可能执行顺序不同）：

FROM --> WHERE --> GROUP BY --> HAVING --> SELECT --> ORDER BY

6. 复杂查询

6.1 视图

视图可以理解为虚拟的表，只包含使用时动态检索数据的查询。表中存储的是实际数据，而视图中保存的是从表中取出数据所使用的SELECT语句。

视图的优点：

  • 由于视图无需保存数据，可以节省存储设备的容量。
  • 将经常使用的SELECT语句做成视图，不用每次都重新书写。而且，视图中的数据会随着原表的变化而自动更新。

创建视图：

CREATE VIEW 视图名称(<视图列名1>, <视图列名2>, ...)
AS
<SELECT 语句>

使用视图的查询：首先执行定义视图的SELECT语句，然后根据执行的结果，再执行FROM子句中使用视图的SELECT语句。

注意：多重视图会降低SQL的性能，我们应该避免在视图的基础上创建视图。

删除视图：

DROP VIEW 视图名称(<视图列名1>, <视图列名2>, ...)

6.2 子查询

子查询可以理解为一次性视图，就是将用来定义视图的SELECT语句直接用于FROM子句中。

示例：

SELECT product_type, cnt_product
FROM (SELECT product_type, COUNT(*) AS cnt_product
      FROM Product
      GROUP BY product_type) ProductSum;

FROM后的为子查询，原则上子查询必须设定名称。子查询作为内层查询会首先被执行。

标量子查询：就是返回单一值的子查询。

示例：查询出销售单价高于全部商品平均单价的商品

SELECT product_id, product_name, sale_price
FROM Product
WHERE sale_price > (SELECT AVG(sale_price)
                  FROM Product);

标量子查询的书写位置不仅仅局限于WHERE子句中，通常任何可使用单一值的位置都可以使用。也就是说，能够使用常数或者列名的地方，无论是在SELECT子句、GROUP BY子句，还是ORDER BY子句中。

6.3 关联子查询

在细分的组内进行比较时，需要使用关联子查询。

示例：按照商品种类查询出高于平均售价的商品

SELECT product_id, product_name, sale_price
FROM Product AS P1
WHERE sale_price > (SELECT AVG(sale_price)
                   FROM Product AS P2
                   WHERE P1.product_type = P2.product_type);

7. 集合运算

7.1 表的加减法

7.1.1 表的加法–UNION

SELECT <col_name1>, <col_name2>
FROM <tbl_name1>
UNION
SELECT <col_name1>, <col_name2>
FROM <tbl_name2>;

UNION集合运算会除去重复的记录。如果想保留重复行，只需要在UNION后添加ALL即可。

集合运算的注意事项：

  • 作为运算对象的记录的列数必须相同
  • 作为运算对象的记录中列的类型必须一致
  • 可以使用任何SELECT语句，但ORDER BY子句只能在最后一次使用

7.1.2 选取表中公共部分–INTERSECT

SELECT <col_name1>, <col_name2>
FROM <tbl_name1>
INTERSECT
SELECT <col_name1>, <col_name2>
FROM <tbl_name2>
ORDER BY <col_name1>;

如果希望保留重复行，需要使用INTERSECT ALL。

7.1.3 记录的减法–EXCEPT

SELECT <col_name1>, <col_name2>
FROM <tbl_name1>
EXCEPT
SELECT <col_name1>, <col_name2>
FROM <tbl_name2>
ORDER BY <col_name1>;

7.2 联结表

7.2.1 内联结–INNER JOIN

SELECT *
FROM <tbl_name1> tn1
INNER JOIN <tbl_name2> tn2
ON tn1.<col_name> = tn2.<col_name>;

7.2.2 外联结–OUTER JOIN

  • 左联结–LEFT JOIN
SELECT *
FROM <tbl_name1> tn1
LEFT JOIN <tbl_name2> tn2
ON tn1.<col_name> = tn2.<col_name>;
  • 右联结–RIGHT JOIN
SELECT *
FROM <tbl_name1> tn1
RIGHT JOIN <tbl_name2> tn2
ON tn1.<col_name> = tn2.<col_name>;

外联结中使用LEFT、RIGHT来指定主表。

8. 函数与CASE表达式

8.1 函数

介绍MySQL支持的一些函数。

8.1.1 算术函数

  • ABS(x)：绝对值
  • CEIL(x)：向上取整，返回大于或等于 x 的最小整数
  • FLOOR(x)：向下取整，返回返回小于或等于 x 的最大整数
  • ROUND(x)：四舍五入函数
  • MOD(x)：取模函数

8.1.2 字符串函数

  • CONCAT(s1,s2…sn)：将多个字符串合并为一个字符串
  • FIND_IN_SET(s1,s2)：返回在字符串s2中与s1匹配的字符串的位置
  • LOCATE(s1,s)：从字符串 s 中获取 s1 的开始位置
  • LOWER(s)/UPPER(s)：将字符串转换为小写/大写
  • LENGTH(s)：返回字符串长度
  • REPLACE(s,s1,s2)：将对象字符串s中的字符串s1替换为s2
  • SUBSTRING(s, start, length)：从字符串 s 的 start 位置截取长度为 length 的子字符串

8.1.3 日期函数

  • ADDDATE(d,n)：计算起始日期 d 加上 n 天的日期
  • CURDATE()/CURRENT_DATE()：返回当前日期
  • DATEDIFF(d1,d2)：计算日期 d1与d2 之间相隔的天数
  • CURRENT_TIMESTAMP()：返回当前日期和时间
  • CURRENT_TIME()/CURTIME()：返回当前时间
  • DATE()：从日期或日期时间表达式中提取日期值
  • DATE_FORMAT(d, f)：按表达式 f 的要求显示日期 d，如DATE_FORMAT(d, “%Y-%m-%d”)
  • DATE_ADD(d，INTERVAL expr type)：计算起始日期 d 加上一个时间段后的日期，如type默认为天，可指定为MINUTE等
  • DATE_SUB(date,INTERVAL expr type)：从日期减去指定的时间间隔expr
  • MONTH(d)/DAY(d)/HOUR(t)/MINUTE(t)/SECOND(t)：返回月份/天/小时/分钟/秒
  • NOW()：返回当前日期和时间
  • WEEKDAY(d)：返回日期 d 是星期几

8.2 CASE表达式

CASE WHEN <求值表达式> THEN <表达式>
     WHEN <求值表达式> THEN <表达式>
     ...
     ELSE <表达式>
END

CASE表达式后的END是不能省略的。
