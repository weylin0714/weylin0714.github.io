1. Node.js安装配置

方式一：

  • 下载源码，下载链接：https://nodejs.org/en/download/ ，这里我下载了最新版本：v12.16.0。
# 进入到/usr/local/src目录下，执行下面命令，在线下载nodejs。
cd /usr/local/src
wget https://nodejs.org/dist/v12.16.0/node-v12.16.0.tar.gz
  • 解压源码
# 解压源码
tar -zxvf node-v12.16.0.tar.gz
  • 编译安装
cd node-v12.16.0
./configure --prefix=/usr/local/node
make
make install
  • 配置环境变量
# 编辑配置文件
vim /etc/profile
# 设置环境变量
export NODE_HOME=/usr/local/node
export PATH=$NODE_HOME/bin:$PATH

上述这种方式，在编译环节比较耗时，我按照下面方式安装，使用官网提供的 二进制包进行安装。

方式二：

  • 下载安装包，下载地址：https://nodejs.org/dist/v12.16.0/node-v12.16.0-linux-x64.tar.gz
# 进入到/usr/local/src目录下，执行下面命令,下载编译好的安装包
wget https://nodejs.org/dist/v12.16.0/node-v12.16.0-linux-x64.tar.gz
  • 解压到/usr/local/目录下，并重命名为nodejs
tar -zxvf node-v12.16.0-linux-x64.tar.gz -C /usr/local/
# 进入到/usr/local目录下，重命名
mv node-v12.16.0-linux-x64 nodejs
  • 添加软连接
ln -s /usr/local/nodejs/bin/node /usr/bin/node
ln -s /usr/local/nodejs/bin/npm  /usr/bin/npm
  • 验证
node -v
npm -v
  • npm更换为淘宝镜像源

npm安装插件是从国外服务器下载，受网络影响大，速度慢且可能出现异常。因此，我们使用淘宝镜像源下载插件。

npm install -g cnpm --registry=https://registry.npm.taobao.org
# 建立软连接
ln -s /usr/local/nodejs/bin/cnpm  /usr/bin/cnpm
# 查看cnpm
cnpm -v
  • 安装yarn

Yarn 是为了弥补npm的一些缺陷而出现，可以使用yarn相关命令进行编译。

# 使用shell脚本安装yarn
curl -o- -L https://yarnpkg.com/install.sh | bash
# 配置使用yarn国内的源
yarn config set registry https://registry.npm.taobao.org

2. Superset开发环境配置

Superset开发环境配置在CentOS7，代码开发主要在Win10进行，采用Pycharm作为开发IDE，远程连接到CentOS7中配置的虚拟Python环境。

2.1 安装系统依赖

安装以下Superset所需的系统依赖模块。

yum install gcc gcc-c++ libffi-devel python-devel python-pip python-wheel openssl-devel cyrus-sasl-devel openldap-devel

2.2 创建Python虚拟环境。

pip install virtualenv
# 在/usr/local/目录下创建python虚拟环境
virtalenv superset_env
# 虚拟环境激活与退出
source superset_env/bin/activate
deactivate

2.3 下载Superset源码

  • 从github上克隆Superset代码到本地，并检出指定版本，我们选择的版本为：0.25.6
git clone https://github.com/apache/incubator-superset
git checkout 57e0564  # Github中版本0.28.1对应的SHA
  • Pycharm打开Superset项目文件

打开项目文件，并配置Pycharm的Project Interpreter到远程服务器上创建的虚拟环境，并配置文件映射路径。比如，我的Python解释器路径为/usr/local/superset_env/bin/python，项目文件存放在/opt/incubator-superset中。

2.4 修改元数据库

Superset默认元数据库是SQLite，实际生产环境中可能需要更可靠的数据库（如MySQL、Oracle）来存储管理元数据。在编译安装前，我们先修改用MySQL替换默认的SQLite。

  • 安装依赖
# 安装在Python虚拟环境
pip install mysqlclient
  • 修改superset配置

修改Superset配置文件config.py，我的路径为/opt/incubator-superset/superset/。

# SQLALCHEMY_DATABASE_URI = 'sqlite:///' + os.path.join(DATA_DIR, 'superset.db')
SQLALCHEMY_DATABASE_URI = 'mysql://db_user:password@db_hostname/superset?charset=utf8'

db_user，password分别为MySQL数据库的用户名和密码，db_hostname为主机IP，superset为存储元数据的数据库，需要创建。

  • 创建superset数据库

在MySQL中创建superset数据库，存储Superset的元数据。

mysql -uroot -p   # 进入MySQL命令行
create database superset;
use superset;
alter database superset character set utf8;

2.5 编译安装与初始化

  • 编译

激活Python虚拟环境，切换到/opt/incubator-superset/superset/assets目录下，执行以下命令进行编译。

(superset_env) [root@centos assets]# yarn
(superset_env) [root@centos assets]# yarn run build
  • 安装

在安装过程中会安装各种Python依赖包，可能会报错，主要是依赖包版本问题或者找不到依赖包，需要针对报错提示进行安装。

# 切换到/opt/incubator-superset/目录下
(superset_env) [root@centos incubator-superset]# python setup.py install

可能会报错：Not a directory: ‘../superset/static/assets/package.json’

解决办法：删除”../superset/static/“目录下的assets文件，并将”../superset/“下的assets文件夹复制到”../superset/static/“目录下。

  • 初始化
# 创建创建管理员用户，根据提示输如用户名、邮箱、密码
(superset_env) [root@centos incubator-superset]# fabmanager create-admin --app superset
# 初始化数据库
(superset_env) [root@centos incubator-superset]# superset db upgrade
# 加载示例数据
(superset_env) [root@centos incubator-superset]# superset load_examples
# 初始化角色和权限
(superset_env) [root@centos incubator-superset]# superset init

2.6 启动Superset

可以采用以下两种方式启动。

方式一：在项目文件incubator-superset/superset/目录下新建run.py文件，文件内容为：

from superset import app
app.run(debug=True, host='0.0.0.0', port=8088)

Pycharm运行run.py文件，访问Superset。由于我的Superset环境部署在CentOS服务器上，在本地访问：IP:8088。

方式二：命令行启动

(superset_env) [root@centos incubator-superset]# superset runserver -d
